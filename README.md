# 比特币助记词生成器

完全离线的 BIP39 助记词生成工具，支持比特币地址生成（P2WPKH/SegWit 格式）。

## ✨ 主要特性

- ✅ **完全离线运行** - 无需网络连接，保护隐私安全
- ✅ **内置 BIP39 词表** - 无需外部文件，开箱即用
- ✅ **标准兼容** - 遵循 BIP39、BIP32、BIP84 标准
- ✅ **比特币地址生成** - 自动生成 P2WPKH（SegWit/Bech32）格式收款地址
- ✅ **加密安全随机数** - 使用浏览器原生 Web Crypto API
- ✅ **私钥安全保护** - 实施主动清零机制，敏感数据立即清除
- ✅ **日/夜主题切换** - 支持白天和夜晚两种显示模式
- ✅ **响应式设计** - 完美适配手机、平板和电脑
- ✅ **开源可审计** - 代码完全开源，可自行审查

## 🚀 快速开始

### 方法一：直接使用（推荐）

1. 下载 `hubble-mnemonic-generator2.0.html` 文件
2. 直接在浏览器中打开（双击文件即可）
3. 内置词表会自动加载，无需任何配置
4. 点击"生成助记词"即可开始使用

**注意**：由于内置了完整的 BIP39 词表，即使使用 `file://` 协议打开，也能正常工作。

### 方法二：使用 Web 服务器运行

如果您希望使用外部 `english.txt` 文件（覆盖内置词表）：

1. 将 `hubble-mnemonic-generator2.0.html` 和 `english.txt` 放在同一目录
2. 在该目录打开终端/命令行，运行：

```bash
# Python 3
python -m http.server 8000

# 或 Python 2
python -m SimpleHTTPServer 8000

# 或 Node.js
npx http-server -p 8000
```

3. 在浏览器中访问 `http://localhost:8000/hubble-mnemonic-generator2.0.html`

## 📖 使用说明

### 生成助记词

1. **选择助记词长度**：
   - 点击"12个单词"或"24个单词"按钮
   - 12个单词：128位熵，适用于大多数用户
   - 24个单词：256位熵，更高的安全性

2. **生成助记词**：
   - 点击"生成助记词"按钮
   - 系统会使用加密安全随机数生成助记词
   - 助记词会以醒目的橙色背景卡片形式显示

3. **查看比特币地址**：
   - 生成助记词后，页面会自动显示对应的比特币收款地址
   - 地址格式为 P2WPKH（SegWit/Bech32），以 `bc1` 开头
   - 点击地址旁的"复制"按钮可以复制地址

4. **保存助记词**：
   - ⚠️ **重要**：请立即用纸笔抄写助记词并妥善保管
   - 助记词一旦丢失，将无法找回资产
   - 建议备份多份并存放在不同地点

### 界面功能

- **主题切换**：点击右上角的太阳/月亮图标切换日/夜模式
- **清除内容**：点击"清除"按钮清空当前显示的助记词和地址
- **下载工具**：点击"下载"按钮保存当前工具到本地（下载的文件不包含助记词）

## 🔐 安全特性

### 隐私保护

- ✅ **完全离线**：所有操作在本地完成，不会连接任何服务器
- ✅ **无数据存储**：助记词和私钥不会保存到本地存储（localStorage）
- ✅ **无网络传输**：不会向外部服务器发送任何数据
- ✅ **主动清零**：敏感数据（私钥、种子）在使用后立即清零
- ✅ **无跟踪脚本**：已移除所有外部跟踪和分析脚本

### 安全措施

1. **加密安全随机数生成**
   - 使用 `crypto.getRandomValues()` API
   - 符合 BIP39 标准要求
   - 不依赖不安全的 `Math.random()`

2. **私钥保护机制**
   - 所有私钥变量都是函数局部变量
   - 私钥在使用后立即清零（`.fill(0)`）
   - 即使函数出错，也会清零敏感数据

3. **错误处理优化**
   - 不输出详细错误信息，避免潜在的信息泄露
   - 错误消息是通用的，不包含敏感数据

4. **下载功能安全**
   - 下载的 HTML 文件不包含助记词
   - 下载的 HTML 文件不包含地址信息
   - 使用文档克隆，不影响当前页面显示

### 安全建议

⚠️ **请务必遵循以下安全建议**：

1. **离线使用**：
   - 在完全离线的环境中使用本工具
   - 生成助记词前，建议断开网络连接

2. **环境安全**：
   - 使用前关闭浏览器开发者工具
   - 禁用可疑的浏览器扩展
   - 建议在隐私模式（无痕模式）中使用

3. **及时清理**：
   - 使用完毕后点击"清除"按钮
   - 关闭浏览器标签页
   - 如果可能，清除浏览器缓存

4. **不要截图**：
   - 不要在联网设备上截图包含助记词或地址的页面
   - 不要将助记词存储在任何联网设备上

5. **验证词表**：
   - 如果使用外部 `english.txt` 文件，请验证其 SHA256 校验值
   - 标准 BIP39 词表应包含恰好 2048 个单词

## 📋 技术说明

### 标准支持

- **BIP39**：助记词生成标准
  - 使用 PBKDF2-SHA512，2048 次迭代
  - 支持 12 词（128位熵）和 24 词（256位熵）

- **BIP32**：分层确定性钱包
  - 实现主密钥和子密钥派生
  - 支持硬化派生和非硬化派生

- **BIP84**：Native SegWit 层次结构
  - 使用标准派生路径 `m/84'/0'/0'/0/0`
  - 专门用于 Native SegWit (P2WPKH) 地址
  - 兼容主流比特币钱包

- **BIP173**：Bech32 编码
  - 实现完整的 Bech32 编码算法
  - 生成 P2WPKH（SegWit）地址

### 加密算法

- **SHA-256**：用于熵校验和计算
- **SHA-512**：用于 PBKDF2 密钥派生
- **HMAC-SHA512**：用于 BIP32 密钥派生
- **RIPEMD160**：用于公钥哈希计算
- **secp256k1**：椭圆曲线加密
  - 完整实现椭圆曲线点运算
  - 支持未压缩和压缩公钥格式

### 地址格式

生成的比特币地址格式为 **P2WPKH（Pay-to-Witness-Public-Key-Hash）**：

- **格式**：SegWit/Bech32
- **前缀**：`bc1`（主网）
- **优点**：
  - 交易手续费更低
  - 更安全的隔离见证（SegWit）
  - 兼容主流钱包（Electrum、Ledger、Trezor 等）

### 浏览器兼容性

- ✅ Chrome/Edge（推荐）
- ✅ Firefox
- ✅ Safari
- ✅ Opera
- ✅ 移动端浏览器

**要求**：支持 Web Crypto API 的现代浏览器

## 🛠️ 故障排除

### 问题：页面提示"默认词表加载失败"

**原因**：
- 旧版本需要外部 `english.txt` 文件
- 使用 `file://` 协议时，浏览器 CORS 限制

**解决方案**：
- ✅ **新版本已内置词表，无需外部文件**
- 如果仍需要外部文件，请使用 Web 服务器运行（见方法二）
- 或手动上传 `english.txt` 文件

### 问题：生成的地址无法使用

**检查**：
- 确认地址格式为 `bc1` 开头（P2WPKH）
- 确认使用的钱包支持 SegWit 地址
- 尝试使用 Electrum、BlueWallet 等支持 SegWit 的钱包

### 问题：助记词无法导入钱包

**检查**：
- 确认助记词是完整的 12 或 24 个单词
- 确认单词拼写正确
- 确认使用标准 BIP39 词表
- 某些钱包可能不支持 24 词助记词
